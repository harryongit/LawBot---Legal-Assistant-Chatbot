LawBot API Documentation

RESTful API endpoints for the LawBot chatbot

Chat Endpoints

POST /api/chat/

Send a message to the chatbot and get an AI response

Request:

{
    "message": "What are the basic requirements for filing a lawsuit?"
}

Success Response:

{
    "response": "To file a lawsuit, you typically need...",
    "message_id": 123
}

Error Response:

{
    "error": "Invalid API key",
    "error_type": "auth_error"
}

Message Endpoints

GET /api/messages/

Get all chat messages

Response:

[
    {
        "id": 1,
        "role": "user",
        "content": "Hello",
        "created_at": "2024-01-01T12:00:00Z"
    },
    {
        "id": 2,
        "role": "assistant",
        "content": "Hello! How can I help you today?",
        "created_at": "2024-01-01T12:00:01Z"
    }
]

GET /api/messages/?role=user

Filter messages by role (user or assistant)

GET /api/messages/{id}/

Get a specific message by ID

DELETE /api/messages/{id}/

Delete a specific message

Error Types

  • validation_error: Invalid request data
  • configuration_error: API key not configured
  • auth_error: Invalid API key
  • rate_limit: Rate limit exceeded
  • timeout: Request timed out
  • network_error: Network connectivity issues
  • api_error: General API error

Testing the API

You can test these endpoints using curl, Postman, or any HTTP client:

Example with curl:

# Send a message
curl -X POST http://localhost:8000/api/chat/ \
  -H "Content-Type: application/json" \
  -d '{"message": "What is contract law?"}'

# Get all messages
curl http://localhost:8000/api/messages/

# Get only user messages
curl http://localhost:8000/api/messages/?role=user
